## 问题1: 节点评估失败 (已修复)

### 现象
- 日志显示 "Evaluated 0 nodes"
- 节点列表一直显示"评估中..."

### 根本原因
Mihomo (Clash Party) 使用的是 **Unix socket** (`/tmp/mihomo-party-501-1574.sock`)，
而不是 TCP 端口 9090。

原代码只支持 HTTP TCP 连接：
```python
BASE_URL = "http://127.0.0.1:9090"
```

### 修复方案
1. 添加 Unix socket 自动检测
2. 优先使用 `requests-unixsocket` 库
3. 添加 raw socket fallback 实现
4. 添加详细的错误日志

### 代码改动
- `MihomoAPI` 类完全重写
- 添加 `_use_unix_socket()` 检测方法
- 添加 `_raw_socket_request()` 备用实现

## 问题2: 前端调用旧API端点 (已修复)

### 现象
- 日志显示前端还在调用 `/api/current-node` (旧端点)

### 修复方案
已更新 `index.html` 中的JavaScript：
- `/api/current-node` → `/api/current`
- `/api/nodes` (保持不变)
- 新增 `/api/switch/{node_name}`
- 新增 `/api/history`

## 问题3: 没有带宽显示 (设计如此)

### 现象
用户期望看到带宽显示。

### 根本原因
带宽测试 (`speedtest-cli`)：
1. 耗时太长 (10-30秒)
2. 消耗大量流量
3. 结果不稳定
4. 只能测试当前节点，其他节点无法测试

### 解决方案
**已移除带宽测试**，改为更可靠的指标：
- 延迟 (40% 权重)
- 丢包率 (20% 权重)
- 抖动 (15% 权重)
- 稳定性历史 (15% 权重) - 新增
- 地理偏好 (10% 权重) - 新增

## 问题4: IP位置不显示 (已修复)

### 现象
- 切换节点后IP没有变化
- 不知道当前出口地理位置

### 根本原因
原代码没有实现真实IP检测，只调用了Mihomo API。

### 修复方案
1. 新增 `IPDetector` 类
2. 通过代理端口测试真实出口IP
3. 使用多个IP地理API (ipapi.co, ipwho.is, api.ip.sb)
4. 切换后验证IP是否真正改变

### 新增功能
- 显示国家国旗 emoji
- 显示城市和国家
- 显示ISP信息
- 节点列表显示每个节点的位置

## 问题5: 假切换 (已修复)

### 现象
- Dashboard显示切换成功
- 但实际IP没有变化

### 根本原因
原代码只检查API返回码，不验证实际效果。

### 修复方案
`switch_to_node()` 现在执行：
1. 记录当前IP
2. 调用Mihomo API切换
3. 等待3秒
4. 检测新IP
5. 比较IP是否变化
6. 更新状态
